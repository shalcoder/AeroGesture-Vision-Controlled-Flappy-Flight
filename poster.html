CV FLAPPY BIRD ARCADE

The Challenge

Mechanical gaming inputs often lack the fluidity of natural human movement. Most computer vision games suffer from processing lag that breaks the immersion. The challenge was to create a seamless, sub-15ms response loop that translates physical hand "pinching" into immediate in-game flight simulation, optimized for real-time arcade competition.

Abstract

This project implements a gesture-controlled Flappy Bird arcade system using Mediapipe pose estimation. It features a robust multi-threaded Python architecture that separates frame capture, CV processing, and physics rendering. The system integrates a Flask-based cloud leaderboard, enabling real-time global ranking synchronization while maintaining 60 FPS gameplay on standard hardware.

Architecture Design

Video Intake: Dual-buffer frame capture using OpenCV for zero-lag streaming.
Landmark Processing: MediaPipe Hand Tracking detects Thumb (4) and Index (8) tips.
Vector Mapping: Euclidean distance calculation trigger ($D < 30px$) for Bird Jump event.
Backend: Flask REST API manages SQLite for high-score persistence and dashboard stats.

Novelty

Unlike traditional clones, this system uses a Pinch-to-Fly mechanism that mimics a bird's wing flap. It includes a dynamic Web Dashboard with animated glassmorphism particles, providing a dual-screen experience for players and spectators.

Results

The system successfully logged 100+ play sessions with 0.016s frame timing stability. The CV engine accurately detected gestures across 95% of tested user hand sizes. Real-time leaderboard updates achieved <100ms latency from Game Over to Global Sync.
